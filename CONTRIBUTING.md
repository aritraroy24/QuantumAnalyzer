# Contributing to QuantumAnalyzer

Thanks for your interest in contributing.

## Before You Start

- Check existing issues and open pull requests first.
- For bug reports and feature proposals, use the issue templates in `.github/ISSUE_TEMPLATE/`.
- Keep changes focused and scoped.

## Development Setup

1. Clone the repository.
2. Open `QuantumAnalyzer.sln` in Visual Studio.
3. Build with `Release|x64` (or `Debug|x64` during development).
4. Validate extension behavior in Explorer preview/thumbnail/context-menu flows.

## Pull Request Guidelines

- Use clear commit messages.
- Include a short summary of what changed and why.
- Add or update docs when behavior changes.

## Code Quality

- Prefer small, reviewable PRs.
- Avoid unrelated refactors in functional fixes.
- Keep UI behavior consistent across supported file types.

## Release Notes

- Update `CHANGELOG.md` for user-visible changes.
